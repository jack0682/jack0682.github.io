Before reaching for a full stability analysis it pays to decompose the
end-to-end delay of a perception-control loop into a *budget*: a small
set of line items, each tied to a term in the analysis and each
independently measurable.

A first pass:

- **Sensing delay** $\tau_s$ — from physical event to ready observation.
- **Perception delay** $\tau_p$ — from observation to latent state update.
- **Decision delay** $\tau_d$ — from latent state to control signal.
- **Actuation delay** $\tau_a$ — from control signal to effect on plant.

With this structure the stability margin can be stated per-term,
which is more actionable than a scalar bound.
